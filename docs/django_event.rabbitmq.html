django_event.rabbitmq¶

django_event.rabbitmq.client¶

Base RabbitMQ client module. You basically don’t need to use this module.

class django_event.rabbitmq.client.RabbitMQClient(host=settings.RABBITMQ_HOST, port=settings.RABBITMQ_PORT, virtual_host=settings.RABBITMQ_VIRTUALHOST, username=settings.RABBITMQ_USERNAME, password=settings.RABBITMQ_PASSWORD, exchange_name='direct', exchange_type='direct')[source]¶

Base RabbitMQ asynchronous adapter.

on_channel_open(channel, callback=lambda frame: None)[source]¶

Callback on channel open. It will declare exchanges for messaging. See RabbitMQ docs for more information.

Parameters:
  • channel (Channel) – Opened channel.
  • callback (callable object) – Callback that will be executed after channel open.
connect()[source]¶

Asynchronous connection. Connects to RabbitMQ server and establish non-blocking connection. After connection is established on_connected callback will be executed which will try to reconnect if this function failed.

disconnect()[source]¶

Disconnect method. Call this method after you end with messaging.

on_close(connection, *args, **kwargs)[source]¶

On close callback. You don’t need to manually call this method.

Parameters:

  • args – Internal args
  • kwargs – Internal kwargs
on_connected(connection)[source]¶

Callback on connection. Reconnect if connection dropped. Otherwise it will open channel and on_channel_open callback will be executed.

Parameters:connection (TornadoConnection) – RabbitMQ connection.
reconnect()[source]¶

Reconnect method. Basically you don’t need to call this method manually.
